# Human Node Guide

[English](human-node.md) | [简体中文](human-node_zh-CN.md)

In DevSwarm, a **Human Node** is your primary workspace. It maps to a Git Worktree and a Tmux Session, providing an isolated environment for your manual coding tasks.

## 1. Initialize Workspace

First, create a directory for your project and initialize DevSwarm:

```bash
mkdir my-project_swarm
cd my-project_swarm
devswarm init https://github.com/user/repo.git
```

This command:
- Clones the main repository into `main_repo/`.
- Creates `.devswarm/` configuration directory.
- Installs Git Hooks for workflow automation.

## 2. Spawn a Node

Create a new node to work on a specific feature.

```bash
# Syntax: ds spawn <branch> <node-name> --base <base-branch>
ds spawn feature/login login-node --base main --label "Frontend"
```

- **Branch**: The logical branch you want to work on (e.g., `feature/login`).
- **Node Name**: A unique name for this environment (e.g., `login-node`).
- **--base**: The base branch to branch off from if `feature/login` doesn't exist.

## 3. Enter the Node

Start coding by entering the node's environment:

```bash
ds enter login-node
```

This will:
- Attach you to a dedicated **Tmux Session**.
- Change the directory to the node's **Git Worktree**.
- Isolate your process from other nodes.

## 4. VSCode Integration

DevSwarm automatically maintains a `DevSwarm.code-workspace` file in the root.

1. Open this workspace file in VSCode: `code DevSwarm.code-workspace`.
2. All your active nodes (Worktrees) will appear as root folders in the Explorer.
3. You can edit files across multiple nodes simultaneously in one window.

## 5. Cleaning Up

When you are done with a task:

```bash
ds rm login-node
```

**Safety Check**: If there are unapplied agent workflows (code changes generated by AI that you haven't merged), `ds rm` will warn you to prevent data loss.
